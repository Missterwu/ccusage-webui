# ccusage-webui

**A local web UI for visualizing Claude Code token usage and cost data.**

Built on top of [ccusage](https://github.com/ryoppippi/ccusage) — the excellent CLI tool for tracking Claude Code API usage. This project wraps ccusage with a browser-based dashboard that lets you explore your spending by project, session, model, and time period without touching the command line.

> **中文说明**：本项目基于 [ccusage](https://github.com/ryoppippi/ccusage) 开发，提供一个本地 Web 界面，可视化展示 Claude Code 的 token 用量与费用数据，支持中英文切换。

## Features

- **One-click reports** — daily, weekly, monthly, session breakdowns, billing block overview
- **Custom queries** — filter by date range, project, sort order
- **Rendered HTML dashboards** — structured tables with cost, token, cache, and model breakdowns (not raw terminal output)
- **Auto-export** — every query silently regenerates the full-day report in JSON, HTML, and Markdown
- **History panel** — all past reports listed and accessible, with one click to open
- **i18n** — switch between Chinese and English with a single click, preference saved across sessions
- **Zero dependencies** — pure Node.js, no npm packages required at runtime
- **Auto-discovers ccusage** — finds the binary via `PATH`, common npm global paths, or an env override

## Prerequisites

- **Node.js** ≥ 18
- **[ccusage](https://github.com/ryoppippi/ccusage)** installed and working

```bash

npm install -g ccusage

# Verify it works
ccusage daily
```

## Usage

```bash
ccusage-webui
```

The dashboard opens automatically at `http://localhost:7788`.

### What you can do

| Section | Description |
|---------|-------------|
| **Quick Actions** | One-click buttons: today's report, this week, this month, sessions, billing blocks |
| **Custom Query** | Choose report type, date range, project filter, sort order |
| **Output panel** | Streaming output from ccusage, with a "View Report" button when done |
| **History** | All generated reports listed by date — click to open HTML or Markdown version |

### Language switching

Click the **EN / 中文** button in the top-right corner to toggle between English and Chinese. The preference is saved to `localStorage` and persists across page reloads and report pages.

## Configuration

All settings are via environment variables. None are required.

| Variable | Description | Default |
|----------|-------------|---------|
| `PORT` or `CCUSAGE_PORT` | HTTP port to listen on | `7788` |
| `CCUSAGE_DATA_DIR` | Directory where reports are saved | `~/.ccusage-webui/` |
| `CCUSAGE_BIN` | Path to the ccusage binary | auto-detected |
| `CCUSAGE_PROJECT_ROOT` | Path prefix to strip from project names | `~` (home directory) |

### Example: custom port and data directory

```bash
PORT=8080 CCUSAGE_DATA_DIR=~/Desktop/reports ccusage-webui
```

### `CCUSAGE_PROJECT_ROOT` — cleaner project names

ccusage identifies projects by their full path on disk (e.g. `/Users/you/Projects/myapp`), which gets encoded into a long slug. By default this dashboard strips only the home directory prefix, which can leave names like `Projects-myapp-src`. Set `CCUSAGE_PROJECT_ROOT` to the parent directory of your projects so only the short name remains:

```bash
# If your projects live at ~/Projects/foo, ~/Projects/bar, etc.
export CCUSAGE_PROJECT_ROOT=~/Projects
ccusage-webui
# Project names will show as: foo, bar, ...
```

## Data Directory

Reports are stored in `~/.ccusage-webui/` by default. The structure:

```
~/.ccusage-webui/
├── usage-2026-05-12.json       # Full day data (JSON)
├── usage-2026-05-12.html       # Full day dashboard (HTML)
├── usage-2026-05-12.md         # Full day report (Markdown)
├── report-daily-2026-05-12-14-30.html
├── report-session-2026-05-12-14-31.html
├── report-weekly-2026-05-12-14-32.html
└── ...
```

The `usage-YYYY-MM-DD.html` files are the full structured dashboards generated by the **Export Today** button. Individual `report-*` files are per-query snapshots.

## How It Works

```
Browser → ccusage-webui (Node.js HTTP server)
                 ↓
          ccusage CLI  (--json flag)
                 ↓
          JSON parsed server-side
                 ↓
          HTML rendered and saved
                 ↓
          Text summary streamed to browser output panel
```

The server runs ccusage with `--json` output, parses the structured data, renders it into styled HTML reports, and streams a text summary to the browser output panel. No double execution — one ccusage call per query.

## Report Types

### Full Dashboard (`usage-YYYY-MM-DD.html`)

The richest report, combining daily project breakdown, session details, model breakdown, token composition, and cost distribution — with donut charts and a bar chart.

Generated by:
- The **Export Today** button
- Automatically in the background after every query

### Query Reports (`report-*.html`)

Lightweight structured tables for a single query. Types:

| Type | ccusage command | What it shows |
|------|----------------|---------------|
| Daily | `ccusage daily` | Cost & tokens per project, per day |
| Weekly | `ccusage weekly` | Cost & tokens per week |
| Monthly | `ccusage monthly` | Cost & tokens per month |
| Session | `ccusage session` | Cost & tokens per conversation session |
| Blocks | `ccusage blocks` | Billing window breakdown |
